    搭建一个bt(或pt)服务器

    Jager_Z 2/28/2022 torrent服务器权限

    此文用来记录、备忘transmission服务器的搭建。主要涉及到Linux文件系统权限的知识。

    # 准备

    一个headless的，NAT类型良好的的树莓派。

    # 安装transmission

    sudo apt update
sudo apt upgrade
sudo apt install transmission-daemon
    1
    2
    3

    # 配置

    1. 先停止trasmission-daemon的后台运行，因为如果在其运行时修改配置文件，会被其自动复位，导致修改无效。

      sudo systemctl stop transmission-daemon
      1

    2. 用mkdir创建一个自己想用来作为torrent下载目的地的文件夹。并修改其用户权限：

      sudo chown -R pi:pi /home/pi/你的文件夹目录
      1

      注：这里是说把目录的拥有权交给pi组的pi用户。如果你想用其他组的用户请酌情修改。

    3. 修改transmission的配置文件：

      sudo vim /etc/transmission-daemon/settings.json
      1

      ​ 我认为最需要关注的修改项为：

      1. rpc-password和rpc-username。（这两项是网页控制面板的登陆账号和密码，如果你有一定安全隐私考虑，一定会要修改）（默认密码是哈希码，你可以直接替换成你想要的明文，重新启动transmission后软件会自动生成并替换为新的哈希码👌）
      2. rpc-whitelist。（改为你自己的内网网段，如192.168.0.*，不然控制面板会返回403）

    4. 将修改过的配置文件引用到pi家目录里的配置目录下，因为transmission会使用指定的（非默认）用户家目录下的配置文件，但其并不会自动生成，所以要进行如下配置：

      1. 首先，获取/etc/目录下配置文件的权限：

        sudo chown -R pi:pi /etc/transmission-daemon
        1

      2. 将文件链接到家目录下，并把权限继续分配给pi用户：

        sudo mkdir -p /home/pi/.config/transmission-daemon/
sudo ln -s /etc/transmission-daemon/settings.json /home/pi/.config/transmission-daemon/
sudo chown -R pi:pi /home/pi/.config/transmission-daemon/
        1
        2
        3

    5. 修改系统守护程序的运行用户：

      sudo vim /etc/init.d/transmission-daemon
sudo vim /etc/systemd/system/multi-user.target.wants/transmission-daemon.service
      1
      2

      分别修改上述文件里user(USER)字段的值，改为pi（或你之前修改目录权限时所指定的用户）

      修改结束之后，重新加载systemctl配置，并启动守护程序

      sudo systemctl daemon-reload
sudo systemctl start transmission-daemon
      1
      2

    6. 使用网页控制面板进一步配置：

      1. 登陆网址为树莓派的ip➕/etc/transmission-daemon中的端口号（默认为9091）
      2. 输入/etc/transmission-daemon中的用户名和密码以登陆
      3. 点击左下角的🔧标志来配置，将下载目录修改为树莓派之前指定的下载目录
      4. bt的话在peer界面将option全部选上✅，pt则全部不勾选。
      5. 点击左上角，输入链接或者上传种子文件就可开始下载了。下载完之后，服务器会自动做种。

    Enjoy!!!🎉
